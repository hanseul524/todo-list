# Acceptance Test Scenarios: SPEC-TODO-001

## 인증 시나리오

### TC-AUTH-01: 정상 로그인
Given: 이메일 "user@test.com", 비밀번호 "Password1!"로 가입된 사용자가 존재한다
When: 로그인 폼에 정확한 이메일/비밀번호를 입력하고 제출한다
Then: `/` (dashboard)로 리다이렉트되고 투두 목록 페이지가 표시된다

### TC-AUTH-02: 비인증 사용자 보호 라우트 접근 차단
Given: 로그인하지 않은 상태이다
When: `/` (dashboard) URL에 직접 접근을 시도한다
Then: `/login`으로 자동 리다이렉트된다

### TC-AUTH-03: 잘못된 비밀번호로 로그인 시도
Given: 등록된 이메일에 잘못된 비밀번호를 입력한다
When: 로그인 폼을 제출한다
Then: 페이지는 유지되고 Toast 에러 메시지가 표시된다

### TC-AUTH-04: 중복 이메일 회원가입 시도
Given: 이미 가입된 이메일로 회원가입을 시도한다
When: 회원가입 폼을 제출한다
Then: Toast 에러 "이미 사용 중인 이메일입니다" 메시지가 표시된다

### TC-AUTH-05: 로그아웃
Given: 로그인된 사용자이다
When: 헤더 유저 메뉴에서 로그아웃을 클릭한다
Then: 세션이 종료되고 `/login`으로 리다이렉트된다

## 카테고리 시나리오

### TC-CAT-01: 카테고리 생성
Given: 로그인된 사용자이다
When: 카테고리 이름 "업무", 색상 `#5e6ad2`로 카테고리를 생성한다
Then: 카테고리 목록에 즉시 추가된다 (낙관적)

### TC-CAT-02: 카테고리 삭제 — 연결된 투두 유지
Given: "업무" 카테고리에 연결된 투두 3개가 존재한다
When: "업무" 카테고리를 삭제한다
Then: 투두 3개는 그대로 유지되고, 해당 투두들의 카테고리 Badge만 제거된다 (`category_id = null`)

### TC-CAT-03: 투두에 카테고리 Badge 표시
Given: "업무" 카테고리(`color: #5e6ad2`)에 연결된 투두가 존재한다
When: 투두 목록을 조회한다
Then: 투두 카드에 "업무" Badge가 표시되며, Badge 색상은 `#5e6ad2` 텍스트 + `#5e6ad2` 20% opacity 배경이다

## 검색 및 필터 시나리오

### TC-FILTER-01: 실시간 제목 검색
Given: "프로젝트 완료", "미팅 준비", "운동하기" 3개 투두가 존재한다
When: 검색창에 "프로"를 입력한다
Then: "프로젝트 완료" 투두만 표시되고 나머지는 숨겨진다 (서버 요청 없음)

### TC-FILTER-02: 완료 필터
Given: 완료 투두 2개, 미완료 투두 3개가 존재한다
When: 필터를 "완료"로 선택한다
Then: 완료 투두 2개만 표시된다

### TC-FILTER-03: 카테고리 필터
Given: "업무" 카테고리 투두 2개, 카테고리 없음 투두 3개가 존재한다
When: 카테고리 필터에서 "업무"를 선택한다
Then: "업무" 카테고리 투두 2개만 표시된다

### TC-FILTER-04: 마감일 정렬
Given: 마감일이 각각 2026-06-20, 2026-06-25, 미설정인 투두가 존재한다
When: 정렬을 "마감일순"으로 선택한다
Then: 2026-06-20 → 2026-06-25 → 마감일 없음 순으로 표시된다

### TC-FILTER-05: 우선순위 정렬
Given: 높음, 중간, 낮음 우선순위 투두가 각각 존재한다
When: 정렬을 "우선순위순"으로 선택한다
Then: 높음 → 중간 → 낮음 순으로 표시된다

### TC-FILTER-06: 필터/정렬 새로고침 후 유지
Given: 필터 "미완료", 정렬 "마감일순"을 선택한 상태이다
When: 페이지를 새로고침한다
Then: 동일한 필터("미완료")와 정렬("마감일순") 상태가 유지된다 (Zustand persist)

## 드래그 앤 드롭 시나리오

### TC-DND-01: 수동 정렬 선택 시 DnD 활성화
Given: 정렬 기준이 "생성일순"으로 설정되어 있다
When: 정렬 기준을 "수동 정렬"로 변경한다
Then: 투두 카드에 드래그 핸들(⠿)이 나타나고 dnd-kit이 활성화된다

### TC-DND-02: 수동 정렬 외 기준에서 DnD 비활성화
Given: 정렬 기준이 "마감일순"으로 설정되어 있다
When: 투두 카드에 커서를 올린다
Then: 드래그 핸들이 표시되지 않는다

### TC-DND-03: 투두 순서 변경
Given: 정렬 기준 "수동 정렬", 투두 A(position:1), B(position:2), C(position:3) 순으로 표시된다
When: C를 A 위로 드래그 앤 드롭한다
Then: 즉시 C-A-B 순서로 재배치되고 DB의 position 값이 업데이트된다

### TC-DND-04: 드래그 중 시각 피드백
Given: 수동 정렬 모드이다
When: 투두 카드를 드래그한다
Then: 드래그 중인 카드는 `#5e6ad2` 테두리 + opacity 80%로 표시되고, 드롭 가능 위치에 `#5e6ad2` 2px 가로선이 나타난다
